QUESTÃO 1.1: Por que a acurácia é uma boa métrica para este dataset balanceado?

Resposta: Com um dataset balanceado (aproximandamente 50% de cada classe), a acurácia é uma métrica confiável porque cada classe contribui igualmente para o desempenho geral. Um classificador aleatório acertaria ~50%, então qualquer valor acima disso reflete aprendizado real. Nesse cenário, errar em qualquer uma das classes tem o mesmo impacto, então a média simples de acertos (acurácia) representa bem a qualidade do modelo.

QUESTÃO 1.2: Por que a acurácia é enganosa neste caso? Qual métrica seria mais apropriada?

Resposta: Com 99% de amostras da classe 0, um classificador trivial que sempre prediz classe 0 alcança ~99% de acurácia sem ter aprendido absolutamente nada — ele jamais detecta um caso positivo. Isso é enganoso porque a classe positiva (minoritária) é frequentemente a mais crítica (ex: fraude, câncer, falha de equipamento). As métricas mais apropriadas são:

  • Recall (Sensibilidade): Mede quantos casos positivos reais foram detectados — essencial quando falsos negativos são perigosos.
  • Precision: Mede quantas das predições positivas estão corretas — importante quando falsos positivos têm alto custo.
  • F1-Score: Média harmônica entre Precision e Recall — equilibra ambos os aspectos.
  • AUC-ROC: Mede a capacidade discriminativa do modelo independentemente do limiar de decisão.

QUESTÃO 1.4: O que significa ter um AUC próximo de 1.0? E próximo de 0.5?

Resposta:

  • AUC próximo de 1.0: O modelo tem excelente poder discriminativo — consegue separar perfeitamente as classes positiva e negativa. Para qualquer limiar de decisão, a taxa de verdadeiros positivos (TPR) é muito maior que a taxa de falsos positivos (FPR). O modelo rankeia todos os positivos antes de todos os negativos.
  • AUC próximo de 0.5: O modelo é equivalente a uma classificação aleatória (linha diagonal no gráfico ROC). Ele não tem capacidade discriminativa — suas predicoes são essencialmente aleatórias, sem distinguir as classes.
  • AUC < 0.5: O modelo é pior que aleatório (pode-se inverter suas predicoes para melhorá-lo).

QUESTÃO 2.2: Por que o ROUGE foca em recall enquanto BLEU foca em precision?

Resposta: As duas métricas foram criadas para tarefas com objetivos distintos:

  • BLEU (precision): Desenvolvido para tradução automática. O objetivo principal é garantir que as palavras geradas sejam corretas e estejam na referência — evitar "invenções" (palavras que não deveriam estar lá). Por isso, mede precision: quantas palavras do candidato aparecem na referência.
  • ROUGE (recall): Desenvolvido para sumarização. O objetivo principal é garantir que as informações importantes da referência apareçam no resumo gerado — evitar "omissões". Por isso, mede recall: quantas palavras da referência aparecem no candidato.

QUESTÃO 2.3: Compare os scores BLEU e METEOR para o Modelo B. Por que o METEOR é mais alto?

Resposta: O Modelo B ("Um felino repousa sobre o carpete") usa sinônimos em relação à referência ("O gato está sobre o tapete"):

  • "felino" → sinônimo de "gato"
  • "repousa" → similar a "está"
  • "carpete" → sinônimo de "tapete"

BLEU considera apenas correspondência lexical exata de n-gramas. Como nenhuma dessas palavras substitutivas aparece literalmente na referência, o BLEU é muito baixo (~0.086).

METEOR vai além da correspondência exata: usa stemming (radicais comuns) e consulta ao WordNet para identificar sinônimos e paráfrases. Assim, reconhece equivalências semânticas entre os termos, resultando em um score mais alto (~0.31) que reflete melhor a similaridade real entre as frases.

QUESTÃO 3.1: Compare BLEU vs BERTScore para o Modelo B. O que você observa?

Exercício 3.3: Perplexidade (Conceitual)¶

A perplexidade mede a "confusão" do modelo ao prever o próximo token.

In [16]:
# Simulação conceitual de perplexidade
# PPL = exp(cross_entropy)

# Exemplo: modelo confiante vs incerto
import math

# Modelo confiante: alta probabilidade para o token correto
prob_confiante = [0.8, 0.1, 0.05, 0.05]  # probabilidades para 4 possíveis tokens
# Modelo incerto: probabilidades mais uniformes
prob_incerto = [0.3, 0.25, 0.25, 0.2]

def calcular_perplexidade(probs):
    """Calcula perplexidade baseada em distribuição de probabilidades"""
    entropy = -sum([p * math.log(p) if p > 0 else 0 for p in probs])
    return math.exp(entropy)

ppl_confiante = calcular_perplexidade(prob_confiante)
ppl_incerto = calcular_perplexidade(prob_incerto)

print("=" * 60)
print("PERPLEXIDADE (PPL)")
print("=" * 60)
print(f"Modelo Confiante:")
print(f"  Probabilidades: {prob_confiante}")
print(f"  Perplexidade:   {ppl_confiante:.2f}")

def avaliar_resumo(referencia, candidato):
    """Avalia um resumo usando múltiplas métricas"""
    resultados = {}
    
    # BLEU
    ref_tokens = referencia.lower().split()
    cand_tokens = candidato.lower().split()
    smoothing = SmoothingFunction().method1
    resultados['BLEU'] = sentence_bleu([ref_tokens], cand_tokens, smoothing_function=smoothing)
    
    # ROUGE
    scorer = rouge_scorer.RougeScorer(['rouge1', 'rouge2', 'rougeL'], use_stemmer=True)
    rouge_scores = scorer.score(referencia, candidato)
    resultados['ROUGE-1'] = rouge_scores['rouge1'].fmeasure
    resultados['ROUGE-2'] = rouge_scores['rouge2'].fmeasure
    resultados['ROUGE-L'] = rouge_scores['rougeL'].fmeasure
    
    # BERTScore
    P, R, F1 = score([candidato], [referencia], lang='pt', verbose=False)
    resultados['BERTScore'] = F1[0].item()
    
    return resultados

QUESTÃO FINAL:

1. Qual modelo teve melhor desempenho geral?

O Modelo 1 tende a ter melhor desempenho nas métricas lexicais (BLEU e ROUGE), pois usa palavras mais próximas das referências (ex: "1.1°C", "emissões", "deep learning"). O Modelo 2 pode compensar com BERTScore mais competitivo, por preservar o significado com paráfrases. No geral, o Modelo 1 é superior por manter tanto a fidelidade lexical quanto o sentido semântico.

2. Houve discrepância entre as métricas lexicais (BLEU/ROUGE) e semânticas (BERTScore)?

Sim. O Modelo 2 usa paráfrases válidas ("cortar emissões" em vez de "reduzir emissões", "desafio climático" em vez de "aquecimento global") que BLEU e ROUGE penalizam por não serem correspondências exatas, mas que o BERTScore reconhece como semanticamente equivalentes. Isso demonstra a limitação das métricas lexicais para capturar a qualidade real de resumos.

3. Qual métrica você consideraria mais importante para avaliar resumos? Por quê?

Para avaliar resumos, o BERTScore é a métrica mais importante, pois o objetivo de um resumo é preservar o significado do texto original, não reproduzir palavras exatas. Um bom resumo pode — e deve — usar paráfrases e sinônimos para ser mais conciso. BLEU e ROUGE são complementares e úteis para verificar a fidelidade terminológica (especialmente em domínios técnicos), mas não devem ser a métrica principal. A combinação ideal é BERTScore como métrica principal + ROUGE-L como complementar, para equilibrar semântica e cobertura de conteúdo.
